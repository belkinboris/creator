## Деплой (Timeweb Cloud, отдельное приложение — НЕ смешивать с Аналитиком)
Волна 3 миграции РФ-инфраструктуры (после АвтоПоста и Компаса) — плейбук
в rf-migration.md. Проект переехал с Railway; исторические Railway-шаги
оставлены ниже как справка на случай отката, актуальный путь — Timeweb.

1. Новый репозиторий на GitHub из этой папки.
2. Timeweb Cloud → Приложения → Создать → Backend/Python, **Python 3.12**
   (НЕ 3.14 — ловили проблемы сборки на прошлых волнах).
3. Приватную сеть подключить СРАЗУ при создании приложения — добавить её
   к уже существующему приложению нельзя, придётся пересоздавать.
4. Start command: `uvicorn app.main:app --host 0.0.0.0 --port 8080`
   (не `$PORT` — Timeweb, в отличие от Railway, порт через переменную не
   прокидывает).
5. Health check path: `/health` (лёгкий, не трогает БД — не оставлять поле
   пустым, это вызывает рестарт-луп).
6. Managed PostgreSQL: имя базы по умолчанию — `default_db` (не имя
   кластера). Приватное подключение — без `sslmode`, публичное — с
   `sslmode=require`. Пароль базы — только буквы/цифры.
7. Variables:
   - `DATABASE_URL` — строка подключения к Timeweb Postgres
   - `LLM_PROVIDER=yandex` (по умолчанию и так `yandex`; ставить явно
     не обязательно, но не помешает)
   - `YANDEX_API_KEY`, `YANDEX_FOLDER_ID` — отдельная папка Yandex AI
     Studio под Создателя (не общая с АвтоПостом)
   - `SOZDATEL_OWNER_KEY` — перенести значение как есть с Railway
   - `ANTHROPIC_API_KEY` — не обязателен, нужен только для отката на
     `LLM_PROVIDER=anthropic`
   - Проверка спроса (частотности Вордстата + конкуренты) использует
     тот же `YANDEX_API_KEY` + `YANDEX_FOLDER_ID`, что и LLM — отдельного
     токена заводить не нужно. Старый отдельный OAuth-доступ к Вордстату
     (oauth.yandex.ru, заявка в поддержку Директа) Яндекс упразднил —
     функциональность перенесена в Yandex Cloud Search API.
     Сервисному аккаунту нужна роль search-api.webSearch.user (как в
     АвтоПосте) — без неё блок конкурентов и частотности тихо не
     показываются, сервис не падает.
8. После каждого пуша в GitHub — автодеплой ненадёжен: вручную нажать ▶
   на последнем коммите в Деплой и свериться, что хэш коммита обновился.
9. Корневой домен projectsozdatel.ru: на reg.ru завести/обновить A-запись
   на публичный IP Timeweb-приложения (можно наконец — IP Timeweb
   постоянный, в отличие от Railway).
