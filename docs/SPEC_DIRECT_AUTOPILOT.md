# Спека: автопилот Яндекс Директа

## Идея

Сейчас пользователь после оплаты живого теста получает `/guide/direct` — подробную
инструкцию, как самому запустить рекламу в Яндекс Директе. Это рабочий, но не
идеальный путь: часть людей теряется на настройке рекламного кабинета, боится
случайно потратить бюджет не туда, бросает на середине.

Идея автопилота: вместо инструкции — мы сами создаём рекламную кампанию на
проверочную страницу пользователя (`/l/{idea_id}`) от аккаунта владельца через
API Яндекс Директа. Пользователь получает уведомления о старте кампании и
дальнейших цифрах — и красивую графику на странице проекта `/p/{idea_id}`
(уже есть, см. `static/project.html`). В Директ пользователь вообще не заходит.

Это качественный скачок в продукте: конкуренты просят человека разбираться в
рекламном кабинете, а Создатель берёт это на себя целиком.

## Доступ и права (подтверждено исследованием, 2026-07)

Регистрация приложения — на `oauth.yandex.ru`, раздел **«Яндекс.Директ»**:

- Галочка **«Использовать API Яндекс.Директ» (`direct:api`)** — этого
  достаточно для управления кампаниями СВОЕГО аккаунта. Именно это нужно
  для старта: рекламу всегда крутим с аккаунта владельца на его же
  проверочные страницы, чужие клиентские кабинеты не трогаем.
- Владелец авторизует приложение своим аккаунтом Директа → получает
  OAuth-токен, который идёт в каждый запрос к API (без него API не поймёт,
  чей это кабинет — см. заголовок `Authorization: Bearer <token>`).
- **Агентский режим** (заголовок `Client-Login`, управление чужими клиентскими
  аккаунтами через один агентский логин) — отдельный, более сложный контур.
  Не нужен на старте. Понадобится, если/когда рекламу начнём крутить не со
  своего аккаунта, а от имени пользователей с их бюджетом — вопрос будущего
  масштабирования, не первого запуска.
- **Песочница (sandbox)** доступна сразу, без одобрения Яндекса — на ней
  обкатывается код и структура запросов до подачи заявки на боевой доступ.
  Боевой доступ Яндекс подтверждает отдельно (аналогично тому, как
  подтверждал Wordstat OAuth — см. `app/demand.py`, `/api/diag/yandex`).

## Архитектура (для следующей сессии, когда токен будет на руках)

- Новый модуль `app/direct_api.py`, тот же паттерн `_post`-инъекции для
  тестов, что уже используется в `app/payments.py` и `app/demand.py` —
  никаких новых архитектурных решений, тот же почерк.
- Поля в `SmokeProject` (`app/main.py`): `direct_campaign_id: str = ""`,
  `direct_status: str = ""` (`not_started` / `moderation` / `active` /
  `stopped`).
- Синхронизация статуса кампании — лениво, при открытии `/p/{idea_id}`
  (тот же принцип, что и весь проект: без фоновых воркеров и очередей,
  см. CLAUDE.md «Архитектурные детали»).
- Этапы внедрения:
  1. Песочница — создание тестовой кампании, проверка формата запросов.
  2. Ручной запуск по кнопке владельца в `/desk` (владелец решает, когда
     включить рекламу конкретному проекту).
  3. Автозапуск сразу после оплаты живого теста (`/api/live-test` →
     `paid` → кампания создаётся автоматически).
  4. Уведомления пользователю о старте кампании и текущих цифрах (канал
     уведомлений — вопрос отдельного решения: email, Telegram-бот и т.п.,
     сейчас в проекте нет ни одного из них).

## Экономика (вопрос владельцу, не для этой сессии)

Текущая цена живого теста — 1490 ₽ (`SOZDATEL_LIVE_TEST_PRICE`). Рекламный
бюджет на тест — 3–5 тыс ₽ (см. `static/guide-direct.html`, шаг 1). Это
больше текущей цены. Автопилот не может списывать бюджет ниоткуда — нужно
решить: новый более дорогой тариф «под ключ» (цена + бюджет включены), или
отдельный запрос бюджета к оплате, или наоборот — то что дороже, чем сейчас.
Без этого решения этап 3 (автозапуск после оплаты) не может быть реализован.

## Риски

- Модерация Директа отклоняет некоторые тематики — нужна обработка отказа
  (уведомление владельцу + пользователю, ручной разбор).
- Лимиты API Директа (баллы/запросы в сутки) — при росте количества
  одновременных живых тестов может понадобиться очередь запросов; пока
  объём маленький, не критично.
- Один аккаунт Директа на все кампании — если Яндекс когда-то ограничит
  число активных кампаний на аккаунт, это станет узким местом раньше, чем
  ожидалось.
