# Создатель — CLAUDE.md

## Проект

**Создатель** (projectsozdatel.ru) — сервис проверки бизнес-идей.
Путь пользователя: бесплатная проверка спроса → платный живой тест.

**Стек:** FastAPI + SQLModel + PostgreSQL, Python 3.12
**Запуск:** `uvicorn app.main:app --host 0.0.0.0 --port 8080`
**Healthcheck:** `GET /health` (лёгкий, без обращения к БД)
**Деплой:** GitHub → Timeweb Cloud. Автодеплой ненадёжен — после push владелец жмёт ▶ вручную в панели Timeweb.

Только литералы: `padding: 22px`. Запрещено использовать `var()`, `max()`, `env()` в критичной вёрстке.

## Архитектурные детали

- Приложение — единый FastAPI-процесс, без фоновых воркеров и очередей.
- Все вызовы LLM — `async` через `httpx.AsyncClient`.
- Вердикт (`compute_verdict`) — детерминированная логика, без LLM.
- Rate-limit: 30 событий / 60 сек на IP, in-memory.
- Аутентификация владельца: заголовок `X-Owner-Key` или query-параметр `key`.
- БД по умолчанию: SQLite (dev) / PostgreSQL (production, `DATABASE_URL`).
- Частотность Вордстата пробует ДВА независимых пути по очереди (см.
  `app/demand.py`): официальный Wordstat API (`YANDEX_WORDSTAT_OAUTH_TOKEN`,
  Bearer OAuth) первым, затем прежний прокси внутри Yandex Cloud Search API
  (`YANDEX_API_KEY`/`YANDEX_FOLDER_ID`). Это два разных продукта Яндекса, не
  синонимы. При жалобе «нет данных» — сначала `GET /api/diag/yandex?key=...`
  (owner-only): покажет сырой ответ обоих путей вместо гадания.
- ЮКасса требует чек (54-ФЗ) с `customer.email`/`customer.phone` — контакт
  из формы обязан быть почтой/телефоном для платных заказов (см.
  `payments.valid_receipt_contact`); телеграм-хэндл проходит только в
  заявку без оплаты (без настроенной кассы).
- Личный кабинет покупателя (`/account`) — вход по magic-link на почту, без
  пароля (`app/mailer.py`, SMTP reg.ru Mail-1). Нужны env:
  `SOZDATEL_SMTP_HOST`, `SOZDATEL_SMTP_PORT` (465), `SOZDATEL_SMTP_USER`,
  `SOZDATEL_SMTP_PASSWORD`. `SmokeProject.contact` не проставляется
  автоматически — владелец привязывает вручную через
  `PATCH /api/projects/{idea_id}/contact` при запуске проекта под оплаченную
  заявку живого теста.

## Roadmap

- `docs/SPEC_DIRECT_AUTOPILOT.md` — автопилот рекламы: создавать кампании в
  Яндекс Директе от аккаунта владельца вместо того, чтобы отправлять
  пользователя настраивать рекламу самому. Спека готова, код — когда будет
  на руках OAuth-токен с правом `direct:api`.

## Рабочий цикл

1. **Любое изменение — с тестами.** Добавляешь или меняешь поведение — покрой тестом.
2. **Запускай `python -m pytest tests/` после каждого изменения.** Все тесты должны быть зелёными до сдачи.
3. **Стиль тестов** — смотри `tests/test_sozdatel.py`: `TestClient`, `autouse` fixture для сброса rate-limit, патчинг LLM через `monkeypatch`.
4. **Ветка разработки:** `claude/sozdatel-setup-guidelines-vrx6gq`.
5. **Коммит и push** в конце каждой сессии с осмысленным сообщением.
